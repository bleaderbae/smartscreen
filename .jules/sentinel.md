## 2026-03-01 - Missing Client-Side DoS Protection\n**Vulnerability:** The shopping list allowed arbitrarily long item strings and infinite list items, relying only on localStorage capacity (5MB).\n**Learning:** Even though state is local, uncapped inputs and arrays can lead to memory exhaustion, browser freezing, or corrupted state parsing.\n**Prevention:** Always implement realistic bounds (e.g. `maxLength={50}` for titles, max 100 items for lists) on all arrays and strings stored in persistence, even if there's no backend.
